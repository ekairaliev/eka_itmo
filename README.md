# Chain of Custody CLI

ЛР1, предметная область 5, вариант 3.

Консольное приложение на Java для 2 этапа лабораторной работы по теме `Chain of Custody`.

Программа работает в цикле: пользователь вводит команду, программа выполняет её и ждёт следующую команду до `exit`.

## Возможности

- создание `Sample` через консоль
- просмотр всех созданных `Sample`
- добавление пломбы `Seal` к образцу
- создание события передачи `CustodyEvent`
- просмотр истории передачи образца
- перевод образца в `ON_HOLD` и обратно в `ACTIVE`
- проверка текущего владельца образца
- обработка ошибок без падения программы

## Структура проекта

В проекте используются 4 основные части:

- `model` — классы предметной области
- `validation` — проверка пользовательского ввода
- `service` — работа с коллекциями и генерация `id`
- `cli` — разбор команд и работа с консолью

## Сборка

```bash
mvn package
```

## Запуск

```bash
java -cp target/classes ru.itmo.ekairaliev.cli.Main
```

## Пользовательский ввод

После запуска программа ждёт команды в консоли:

```text
> help
```

Важно:

- угловые скобки в командах вводить не нужно
- правильно: `seal_add 1`
- неправильно: `seal_add <1>`

## Команды

### Общие команды

- `help` — показать список команд
- `exit` — завершить программу

### Команды предметной области

- `sample_add`
- `sample_list`
- `seal_add <sample_id>`
- `seal_show <seal_id>`
- `seal_break <seal_id>`
- `cust_add <sample_id>`
- `cust_list <sample_id> [--last N]`
- `cust_show <event_id>`
- `sample_hold <sample_id>`
- `sample_release <sample_id>`
- `cust_check <sample_id>`
- `cust_export <sample_id>`

## Особенности ввода

### `sample_add`

Интерактивная команда создания образца:

```text
> sample_add
Название sample: Blood Tube #1
OK sample_id=1
```

### `cust_add`

Интерактивная команда создания события передачи:

- `От кого` — только имя, без цифр
- `Кому` — только имя, без цифр
- `Место` — обычный текст
- `Комментарий` — можно оставить пустым

Пример:

```text
> cust_add 1
От кого (только имя): Ivan
Кому (только имя): Petr
Место: Lab room
Комментарий (можно пусто): delivered
OK event_id=1
```

## Обработка ошибок

Программа не должна падать при неправильном вводе.
Обрабатываются случаи:

- неизвестная команда
- неверное число аргументов
- `id` введён не числом
- объект с таким `id` не найден
- ошибки валидации полей

Примеры:

```text
> seal_add abc
Ошибка: sample_id должен быть числом

> cust_show 100
Ошибка: custody_event с id=100 не найден

> cust_add 1
От кого (только имя): Ivan1
Кому (только имя): Petr
Место: Lab room
Комментарий (можно пусто): ok
Ошибка: from должно содержать только буквы
```
